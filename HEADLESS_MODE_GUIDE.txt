🖥️ NEW FEATURE: Control Whether Browser Window Shows!

You can now choose whether the Playwright browser window appears
on your desktop or runs hidden in the background (headless mode).

═══════════════════════════════════════════════════════════════

✅ HOW TO USE:
═══════════════════════════════════════════════════════════════

When opening canvas-scraper.html, you'll see TWO checkboxes:

┌──────────────────────────────────────────────────────────────┐
│ Login to Canvas.io                                           │
│                                                               │
│ Email:    [                          ]                       │
│ Password: [                          ]                       │
│                                                               │
│ ☑ Show browser window (uncheck for headless mode) ← NEW!    │
│ ☑ Show browser preview (live screenshots)                   │
│                                                               │
│ [Start Fetching]                                             │
└──────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════

🎯 CHECKBOX 1: Show browser window
═══════════════════════════════════════════════════════════════

CHECKED ☑ (default): Browser window visible on desktop
UNCHECKED ☐: Headless mode (browser hidden)

═══════════════════════════════════════════════════════════════

🖼️ WHAT'S THE DIFFERENCE?
═══════════════════════════════════════════════════════════════

Browser Window VISIBLE (☑):
───────────────────────────
┌─────────────────────────────────────┐
│ Your Desktop                        │
│                                     │
│  [Your Apps]  [Browser Window] ←── │
│               Shows Canvas.io       │
│               You can see it!       │
└─────────────────────────────────────┘

Benefits:
✓ Visual confirmation
✓ Easy to debug
✓ Educational - see what happens
✓ Can manually intervene if needed

Drawbacks:
✗ Window flashes on screen
✗ Takes focus from other windows
✗ Slightly slower (rendering overhead)
✗ Distracting if working

───────────────────────────────────────────────────────────────

Headless Mode HIDDEN (☐):
──────────────────────────
┌─────────────────────────────────────┐
│ Your Desktop                        │
│                                     │
│  [Your Apps]                        │
│               No browser window     │
│               Runs in background    │
└─────────────────────────────────────┘

Benefits:
✓ No distracting windows
✓ Slightly faster
✓ Can work on other things
✓ Professional/clean execution
✓ Better for automation

Drawbacks:
✗ Can't see what's happening (unless using screenshots)
✗ Harder to debug visually
✗ Must trust the automation

═══════════════════════════════════════════════════════════════

🎛️ TWO CHECKBOXES EXPLAINED:
═══════════════════════════════════════════════════════════════

CHECKBOX 1: Show browser window
→ Controls the ACTUAL browser window on your desktop
→ Playwright's visual vs headless mode

CHECKBOX 2: Show browser preview  
→ Controls SCREENSHOTS in the web interface
→ Visual feedback in your browser

These are INDEPENDENT:
• You can show window but hide preview
• You can hide window but show preview  
• You can show both
• You can hide both

═══════════════════════════════════════════════════════════════

⚡ PERFORMANCE COMPARISON (50 scans):
═══════════════════════════════════════════════════════════════

Window Visible (☑☐): 27-52 seconds
Window Hidden (☐☐):  25-50 seconds

Difference: 2-5 seconds saved with headless mode

Note: Headless is only marginally faster because most time 
      is spent on network operations, not rendering.

🖥️ TECHNICAL DETAILS:
═══════════════════════════════════════════════════════════════

Headless Mode:
• Playwright runs Chromium without GUI
• All functionality intact (can still take screenshots!)
• Slightly less CPU/GPU usage
• No X11/display needed on Linux servers
• Memory usage similar to visible mode

Visible Mode:
• Full Chromium browser with GUI
• Can see clicks, navigation, forms
• Useful for development and debugging
• Takes screen space
• May take focus from other apps

═══════════════════════════════════════════════════════════════

🔧 HOW IT WORKS:
═══════════════════════════════════════════════════════════════

Frontend:
1. User checks/unchecks "Show browser window"
2. JavaScript reads checkbox state
3. Passes to backend as headless=true/false

Backend:
1. Receives headless parameter
2. Uses in chromium.launch({ headless: isHeadless })
3. Browser runs accordingly

Result:
✓ Simple user control
✓ No config files needed
✓ Per-session flexibility

🐛 TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════

Browser window not appearing when checked?
→ Make sure you checked it BEFORE clicking Start
→ Try reloading the page
→ Check backend logs for errors

Headless mode not working?
→ Verify checkbox is unchecked
→ Check browser console for errors
→ Ensure backend received headless=true parameter

Can't see what's happening in headless?
→ Enable browser preview (second checkbox)
→ Or check backend terminal logs
